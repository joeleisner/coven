## 3. `@internal` / `@advanced` Audit

**Rule applied consistently:**

- **`@advanced`** — exported for power users reading raw grimoire state; not part of everyday usage. Applied to all `$*_GRIMOIRE_SYMBOL` exports and their corresponding grimoire type.
- **`@internal`** — not part of the public contract; should not appear as a primary docs entry. Already used on `Familiar.connectedCallback` / `disconnectedCallback`; no new additions.

## 5. New Hex: `$morph`

Completes the observer trio alongside `$mut` (MutationObserver) and `$scry` (IntersectionObserver).

**Name rationale:** ResizeObserver watches things *reshape/morph*. `$morph` is short, immediately readable, and fits the library's magical vocabulary.

### API

```ts
// Grimoire symbol (exported @advanced)
export const $MORPH_GRIMOIRE_SYMBOL: unique symbol;

// Grimoire type (exported @advanced)
export type $MorphGrimoire = {
    /** The set of active ResizeObservers on this element. */
    observers?: Set<ResizeObserver>;
};

// Config
export type $MorphConfig = ResizeObserverOptions & {
    /** Called on each resize observation. */
    callback: ResizeObserverCallback;
};

// Main call — creates a new ResizeObserver each time (intentional;
// different call sites usually want different box options).
$morph(element: HTMLElement, config: $MorphConfig): ResizeObserver

// Sub-methods (read-only, no side effects)
$morph.observers(element): Set<ResizeObserver> | undefined
$morph.disconnect(element): void
```

### Implementation notes

- Mirrors `$scry.ts` exactly: same grimoire pattern, same cleanup via `$bewitch` signal.
- Each call creates a new `ResizeObserver` (same rationale as `$scry` — callers at different call sites typically want independent observers with different `box` options).
- Signal abort disconnects all observers and clears the set.
- `$morph.disconnect` is an escape hatch for eager teardown without aborting the signal.

## 6. Playground: `coven-resize`

Demonstrates `$morph` in the existing Vite playground.

**Behaviour:** A drag-resizable box (`resize: both`) that uses `$morph` to watch its own size. A `<span part="dims">` overlaid in the top-right corner updates live with `W × H` pixel dimensions. Emits `coven-resize:change` events (captured by `coven-log`).
